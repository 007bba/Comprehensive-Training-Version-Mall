 项目成员贡献分工说明

  本项目为前后端分离的综合实训商城系统，整体功能包括用户登录注册、商品展示、商品分类、购物车、订单提交、收货地
  址管理、项目部署与文档整理等。项目采用 Vue 2 作为前端框架，Django REST Framework 作为后端接口框架，使用 JWT
  实现登录认证，并通过数据库保存用户、商品、购物车、订单等业务数据。

  为便于团队协作与成果展示，项目贡献可以划分为以下五个部分。

  ---
  成员一：项目负责人 / 架构设计 / 后端基础搭建

  主要负责项目整体架构设计、后端工程搭建和核心配置。

  具体贡献包括：

  - 负责确定项目整体方向，将项目定位为前后端分离的商城系统。
  - 负责后端 Django 项目基础结构搭建。
  - 负责配置 Django REST Framework 接口框架。
  - 负责配置项目路由、应用注册、中间件和跨域设置。
  - 负责配置 JWT 登录认证相关内容。
  - 负责设计后端基础目录结构，包括 goods、order、users、basic 等业务应用。
  - 负责数据库连接配置和环境变量配置。
  - 负责整体接口风格和模块划分设计。
  - 负责协调前后端接口对接和项目整体进度。

  适合写成一句话：

  ▎ 主要负责项目整体架构设计、后端基础框架搭建、接口规范设计、认证配置和团队开发协调工作。

  ---
  成员二：商品模块 / 首页展示 / 搜索筛选功能

  主要负责商品相关业务模块，包括商品分类、商品列表、商品详情和首页展示。

  具体贡献包括：

  - 负责商品分类模型设计与接口实现。
  - 负责商品模型设计，包括商品名称、分类、价格、库存、销量、点击量、商品详情、主图等字段。
  - 负责首页轮播图模型和接口实现。
  - 负责商品分类导航接口。
  - 负责商品列表接口、商品详情接口。
  - 负责商品搜索、价格筛选、分类筛选、排序和分页功能。
  - 负责前端首页模块开发，包括轮播图、推荐商品、分类商品展示。
  - 负责商品列表页和商品详情页开发。
  - 负责商品详情访问量统计逻辑。
  - 负责商品模块前后端联调。

  适合写成一句话：

  ▎ 主要负责商品分类、商品列表、商品详情、首页轮播、推荐商品、搜索筛选和分页等商品展示相关功能。

  ---
  成员三：用户模块 / 登录注册 / 个人中心 / 地址管理

  主要负责用户体系和用户相关信息管理。

  具体贡献包括：

  - 负责自定义用户模型设计，扩展真实姓名、手机号、性别、生日、头像、会员等级等字段。
  - 负责用户注册接口实现。
  - 负责用户登录认证逻辑。
  - 负责 JWT token 登录流程对接。
  - 负责支持用户名或手机号登录。
  - 负责前端登录页面和注册页面开发。
  - 负责用户信息保存和登录状态维护。
  - 负责个人中心页面开发。
  - 负责用户资料查询和修改功能。
  - 负责收货地址模型设计。
  - 负责收货地址新增、查询、修改、删除和默认地址设置。
  - 负责地址管理页面与后端接口联调。

  适合写成一句话：

  ▎ 主要负责用户注册登录、JWT 认证、个人资料维护、收货地址管理和用户中心相关功能。

  ---
  成员四：购物车模块 / 订单模块 / 支付流程

  主要负责商城核心交易流程，包括购物车、结算和订单。

  具体贡献包括：

  - 负责购物车模型设计。
  - 负责加入购物车接口实现。
  - 负责购物车列表查询、数量修改和删除功能。
  - 负责购物车前端页面开发。
  - 负责使用 Vuex 管理购物车数据、总数量和总价。
  - 负责结算页面开发。
  - 负责订单主表和订单明细表设计。
  - 负责提交订单接口实现。
  - 负责购物车转订单业务流程。
  - 负责订单号生成、订单金额统计、商品数量统计。
  - 负责库存校验、销量更新和购物车清空逻辑。
  - 负责我的订单页面开发。
  - 负责模拟支付或订单支付状态更新流程。
  - 负责订单模块前后端联调。

  适合写成一句话：

  ▎ 主要负责购物车、结算、订单生成、订单明细、库存校验、订单状态和模拟支付等交易流程功能。

  ---
  成员五：前端整合 / 部署运维 / 文档撰写 / 测试展示

  主要负责前端整体整合、项目部署、文档整理和展示材料。

  具体贡献包括：

  - 负责前端公共组件整合，包括头部、底部、导航栏等。
  - 负责前端路由配置和页面跳转逻辑。
  - 负责 Axios 请求封装和接口地址配置。
  - 负责前端页面样式调整和交互优化。
  - 负责项目本地部署流程整理。
  - 负责使用 Conda 管理 Python 环境的部署说明。
  - 负责前端 npm / Node.js 环境配置说明。
  - 负责编写 README.md 部署文档。
  - 负责编写项目架构与功能模块介绍文档。
  - 负责整理项目定位、技术架构、核心流程和展示说明。
  - 负责项目测试、运行截图、演示流程准备。
  - 负责 GitHub 协作说明、提交规范和项目展示材料整理。

  适合写成一句话：

  ▎ 主要负责前端整体整合、部署环境配置、项目文档编写、运行测试和答辩展示材料准备。

  如果要显得每个人工作量更平均，可以这样分

  成员一：后端基础架构 + 用户认证基础
  成员二：商品展示模块 + 首页模块
  成员三：用户中心模块 + 地址模块
  成员四：购物车模块 + 订单模块
