# 搜索功能实现说明书

## 一、功能概述

本项目的搜索功能用于帮助用户在商城中快速查找目标商品。用户可以在页面顶部搜索框输入商品关键词，系统根据关键词跳转到商品列表页，并向后端商品接口发送查询参数，最终展示符合条件的商品结果。

搜索功能属于电商系统的基础功能，主要解决用户在商品数量较多时难以快速定位目标商品的问题。通过搜索框、路由跳转、商品列表页参数承接和后端模糊查询的配合，实现了从用户输入到结果展示的完整闭环。

## 二、实现目标

本次搜索功能实现的目标包括：

1. 用户可以在商城顶部搜索框输入关键词。
2. 点击“搜索”按钮后，页面不会再跳转到旧的静态 `search.php` 页面。
3. 前端通过 Vue 路由跳转到商品列表页。
4. 商品列表页能够读取 URL 中的搜索关键词。
5. 商品列表页将关键词作为参数传递给后端商品接口。
6. 后端根据商品名称、商品描述等字段进行模糊匹配。
7. 前端展示搜索后的商品列表结果。

## 三、原有问题分析

在修复前，项目中已经存在搜索框和部分搜索逻辑，但功能没有真正打通。主要问题集中在前端搜索框表单处。

原搜索框仍保留传统 HTML 表单写法：

```html
<form
  action="search.php"
  method="get"
  id="searchForm"
  name="searchForm"
  onSubmit="return checkSearchForm()"
>
```

这种写法存在以下问题：

1. `action="search.php"` 是旧式页面跳转方式，不适合当前 Vue 单页应用。
2. `onSubmit="return checkSearchForm()"` 中的 `checkSearchForm()` 在当前组件中没有定义。
3. 输入框没有绑定 Vue 数据变量，导致组件方法无法获取用户输入内容。
4. 已经写好的 `doSearch()` 方法没有被表单提交事件调用。

因此，用户点击搜索按钮时，并不会进入 Vue 路由控制的商品列表页，而是尝试执行旧页面表单提交逻辑，导致搜索功能无法正常使用。

## 四、前端实现方案

### 1. 搜索框数据绑定

在头部组件中定义了搜索关键词变量：

```js
data() {
  return {
    searchKeyword: ''
  }
}
```

通过 `v-model` 将输入框内容与 `searchKeyword` 绑定：

```html
<input
  v-model="searchKeyword"
  class="sea_input"
  type="text"
  autocomplete="off"
  placeholder="搜索商品..."
/>
```

这样用户输入内容后，Vue 组件可以实时获取关键词。

### 2. 表单提交事件改造

将原来的静态表单提交改为 Vue 事件提交：

```html
<form id="searchForm" name="searchForm" @submit.prevent="doSearch">
```

其中：

- `@submit.prevent` 表示阻止浏览器默认提交行为。
- `doSearch` 是 Vue 组件中的搜索方法。
- 表单不会跳转到旧的 `search.php`。

### 3. 搜索跳转逻辑

搜索方法如下：

```js
doSearch() {
  const kw = this.searchKeyword.trim()
  if (!kw) return
  this.$router.push({ path: '/list/0', query: { keyword: kw } })
}
```

该方法完成了三件事：

1. 获取用户输入的关键词。
2. 去除关键词前后空格。
3. 如果关键词不为空，则跳转到商品列表页，并将关键词放入 URL query 中。

跳转后的地址示例：

```text
/list/0?keyword=耳机
```

## 五、商品列表页参数承接

商品列表页会读取路由中的 `keyword` 参数：

```js
const keyword = this.$route.query.keyword
if (keyword) {
  params.name = keyword
  params.search = keyword
}
```

然后调用商品接口：

```js
getGoods(params)
```

其中 `params.name` 用于商品名称模糊过滤，`params.search` 用于后端通用搜索过滤。

## 六、前端 API 调用

商品接口封装如下：

```js
export function getGoods(data) {
  return request({
    url: '/goods/',
    method: 'get',
    params: data
  })
}
```

当用户搜索“耳机”时，实际请求类似：

```text
GET /goods/?name=耳机&search=耳机
```

## 七、后端查询过滤

后端商品接口已经配置了过滤和搜索能力。

商品名称过滤通过 `GoodsFilter` 实现：

```python
name = filters.CharFilter(field_name='name', lookup_expr='icontains')
```

该配置表示后端会对商品名称进行模糊匹配。

商品视图中还配置了 DRF 的搜索过滤：

```python
filter_backends = (DjangoFilterBackend, filters.SearchFilter, filters.OrderingFilter)
search_fields = ('name', 'goods_desc')
```

因此后端可以根据商品名称和商品描述进行搜索。

同时，商品列表接口只返回上架商品：

```python
return queryset.filter(status=0)
```

这样可以避免下架商品出现在普通用户搜索结果中。

## 八、完整搜索流程

搜索功能的完整流程如下：

```text
用户在顶部搜索框输入关键词
        ↓
点击“搜索”按钮
        ↓
Vue 表单提交事件触发 doSearch()
        ↓
跳转到 /list/0?keyword=关键词
        ↓
商品列表页读取 keyword 参数
        ↓
调用 /goods/?name=关键词&search=关键词
        ↓
后端根据商品名称和描述进行模糊查询
        ↓
返回符合条件的商品数据
        ↓
前端商品列表页展示搜索结果
```

## 九、测试与验证

功能修复后，进行了页面级验证：

1. 在顶部搜索框输入已有商品关键词。
2. 点击“搜索”按钮。
3. 页面成功跳转到商品列表页。
4. 地址栏出现 `keyword` 查询参数。
5. 商品列表根据关键词展示对应商品。
6. 空关键词不会触发跳转。

验证结果表明，搜索功能已经可以正常使用。

## 十、总结

本次搜索功能修复的核心是将旧式 HTML 表单提交方式改造为 Vue 单页应用中的路由跳转方式。修复后，前端搜索框、商品列表页、商品接口和后端查询过滤形成了完整闭环，实现了商城商品搜索的基础功能。
