# 支付功能实现说明书

## 一、功能概述

支付功能是电商系统交易流程中的核心环节。本项目中的支付功能主要用于模拟用户从购物车或商品购买流程进入订单结算，并完成订单支付状态变更的过程。

综合实训项目中，支付功能重点体现业务流程的完整性，包括：购物车结算、收货地址选择、订单创建、支付确认、订单状态更新和用户订单查看。该功能以模拟支付为主，不接入真实第三方支付平台，适合教学演示和课程答辩场景。

## 二、实现目标

支付功能的主要目标如下：

1. 用户可以将商品加入购物车。
2. 用户可以进入结算页面查看待支付商品。
3. 用户可以选择或填写收货地址。
4. 系统根据购物车商品生成订单。
5. 用户确认支付后，订单状态发生变化。
6. 支付完成后，用户可以在订单页面查看订单。
7. 商家端可以看到订单并进行后续处理。

## 三、业务流程设计

支付功能并不是单独存在的按钮，而是完整交易链路的一部分。整体流程如下：

```text
用户浏览商品
    ↓
加入购物车 / 立即购买
    ↓
进入购物车或结算页
    ↓
确认商品、数量、金额、地址
    ↓
提交订单
    ↓
模拟支付
    ↓
订单状态更新
    ↓
用户查看订单
    ↓
商家处理订单
```

## 四、前端页面实现

### 1. 购物车页面

购物车页面用于展示用户已选择的商品，包括商品名称、图片、单价、数量和总价。用户可以在购物车中调整商品数量，也可以删除不需要购买的商品。

购物车数据通常保存在前端状态管理或后端购物车接口中。结算时，前端会把购物车中的商品带入结算页面。

### 2. 结算页面

结算页面主要负责确认订单信息，包括：

- 商品列表
- 商品数量
- 商品总价
- 收货地址
- 联系方式
- 订单备注

用户确认无误后点击提交订单，前端调用后端订单创建接口。

### 3. 支付确认页面或支付动作

本项目采用模拟支付方式。用户提交订单后，可以通过支付按钮模拟完成支付。支付成功后，前端跳转到订单页面或显示支付成功提示。

示例流程：

```text
点击提交订单
  → 后端创建订单
  → 返回订单 ID
  → 前端发起模拟支付
  → 后端更新订单状态
  → 跳转我的订单页
```

## 五、后端订单与支付逻辑

后端支付功能主要围绕订单状态变化实现。

一般订单状态可以包括：

```text
待支付
已支付 / 待发货
已发货
已完成
已取消
```

模拟支付时，后端不需要真正调用第三方支付平台，而是检查订单是否存在、是否属于当前用户、是否仍处于可支付状态，然后将订单状态更新为已支付或待发货。

后端处理逻辑通常包括：

1. 校验用户身份。
2. 校验订单归属。
3. 校验订单状态是否允许支付。
4. 更新订单支付状态。
5. 记录支付时间或状态变更时间。
6. 返回支付成功结果。

## 六、模拟支付的作用

由于综合实训项目主要强调系统功能完整性和业务流程展示，模拟支付可以避免真实支付接入带来的复杂问题，例如：

- 第三方支付平台资质申请。
- 支付回调签名校验。
- 真实资金流动风险。
- 沙箱环境配置复杂。
- 网络回调地址要求。

通过模拟支付，项目仍然可以完整展示电商交易流程，满足课程实训中的功能演示要求。

## 七、支付功能安全控制

虽然是模拟支付，仍然需要遵守基本安全原则：

1. 不能由前端直接决定订单最终金额。
2. 后端应根据商品价格和购买数量重新计算订单金额。
3. 用户只能支付自己的订单。
4. 已支付订单不能重复支付。
5. 已取消订单不能继续支付。
6. 订单状态变化应由后端统一处理。

这样可以保证即使是模拟支付，也符合真实系统的基本设计思想。

## 八、用户端与商家端联动

支付完成后，订单不只影响用户端，也会影响商家端。

用户端可以查看：

- 订单编号
- 商品信息
- 支付状态
- 收货地址
- 下单时间

商家端可以查看：

- 新订单
- 订单商品
- 买家信息
- 支付状态
- 发货处理入口

因此，支付功能实际上打通了用户购买行为和商家订单管理之间的业务闭环。

## 九、测试与验证

支付功能可以按以下步骤验证：

1. 用户登录系统。
2. 选择商品加入购物车。
3. 进入购物车页面。
4. 点击结算。
5. 在结算页确认商品和地址。
6. 提交订单。
7. 点击模拟支付。
8. 系统提示支付成功。
9. 跳转到我的订单页面。
10. 查看订单状态是否变化。
11. 切换到商家端查看是否能看到该订单。

## 十、总结

本项目的支付功能采用模拟支付方案，重点体现电商系统从商品购买到订单生成、支付确认和商家处理的完整业务流程。该方案不涉及真实资金交易，降低了项目实现难度和演示风险，同时能够清晰展示电商平台的核心交易闭环。
