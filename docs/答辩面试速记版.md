# 商城项目答辩/面试速记版

这份速记版用于答辩、面试或短时间复习。重点不是展开所有细节，而是让你能用清晰的话讲出项目是什么、怎么设计、核心功能怎么实现、你学到了什么。

## 1. 一句话介绍项目

这是一个前后端分离的商城系统，前端使用 Vue 2、Vue Router、Vuex 和 Axios，后端使用 Django REST Framework、MySQL 和 JWT。项目实现了商品展示、分类筛选、商品详情、用户登录注册、购物车、收货地址、订单提交和订单查看等商城核心功能。

## 2. 项目分为哪两部分

项目主要分为前端和后端两部分。

前端目录是 `myshop-vue`，负责页面展示、路由跳转、用户操作、接口调用和全局状态管理。

后端目录是 `myshop-api2`，负责提供 REST API，处理用户、商品、分类、购物车、订单和地址等业务逻辑。

## 3. 前端技术栈怎么说

前端使用 Vue 2 作为主体框架，Vue Router 管理页面路由，Vuex 管理登录用户和购物车状态，Axios 负责请求后端接口，Webpack 负责项目构建。页面主要集中在 `src/views`，接口封装在 `src/api`，请求拦截器在 `src/utils/request.js`，全局状态在 `src/vuex/store.js`。

## 4. 后端技术栈怎么说

后端使用 Django 3.1.5 和 Django REST Framework 搭建接口服务，使用 MySQL 存储数据，使用 JWT 实现登录认证，使用 django-filter 实现商品筛选，使用 DRF 的 ViewSet、Serializer 和 Router 快速开发 REST API。

## 5. 项目核心业务闭环

这个项目的核心业务闭环是：用户进入首页浏览商品分类和推荐商品，进入商品列表页筛选商品，进入商品详情页选择数量并加入购物车，登录后在购物车中修改商品数量，进入结算页选择地址并提交订单，后端根据购物车生成订单和订单明细，最后用户可以在我的订单页面查看订单。

可以背成这一句：

用户浏览商品，加入购物车，确认收货信息，提交订单，后端生成订单并清空购物车，用户最后查看订单。

## 6. 前端路由怎么讲

前端的路由在 `src/router/index.js` 中配置。主要路由包括 `/index` 首页、`/list/:id` 商品列表、`/detail/:id` 商品详情、`/cart` 购物车、`/checkout` 结算页、`/login` 登录、`/reg` 注册、`/profile` 个人资料、`/address` 收货地址、`/myorder` 我的订单。

路由的作用是把不同 URL 映射到对应 Vue 页面组件。页面切换时，项目还会根据路由的 `meta.title` 修改浏览器标题。

## 7. 前端接口调用流程怎么讲

前端页面一般在 `created()` 生命周期中调用接口方法。接口方法写在 `src/api` 目录下，比如商品相关接口在 `api/goods.js`，购物车和订单接口在 `api/order.js`，用户接口在 `api/users.js`，地址接口在 `api/basic.js`。

这些接口最终都会调用 `src/utils/request.js` 中封装好的 Axios 实例。Axios 设置了后端地址 `http://localhost:8000/`，并通过请求拦截器自动给需要登录的接口加上 JWT token。

## 8. 登录认证流程怎么讲

用户在登录页面输入用户名和密码，前端请求后端 `/login/` 接口。后端使用 JWT 登录接口，并通过自定义认证后端支持用户名或手机号登录。

登录成功后，后端返回 token 和用户信息。前端把 token 保存到 `localStorage`，同时把用户信息保存到 Vuex。之后访问购物车、订单、地址、个人资料等需要登录的接口时，Axios 请求拦截器会自动在请求头中添加：

```text
Authorization: JWT <token>
```

如果 token 无效或没有登录，后端返回 401，前端提示用户先登录。

## 9. Vuex 在项目里做什么

Vuex 主要保存两个重要状态：登录用户信息和购物车列表。

登录成功后，用户信息会保存到 Vuex 和本地缓存。退出登录时，Vuex 会清空用户信息、token 和购物车数据。

购物车变化后，前端会重新请求购物车接口，并把最新购物车列表保存到 Vuex。Vuex 中还通过 getter 计算购物车总数量和总价格，所以头部购物车、购物车页面和结算页面都可以共用这些数据。

## 10. 商品模块怎么讲

商品模块主要包括商品分类、商品列表、商品详情和首页轮播。

后端中 `GoodsCategory` 表示商品分类，支持父子分类；`Goods` 表示商品，包含名称、分类、价格、销量、库存、详情、主图等字段；`Slide` 表示首页轮播图。

前端首页会请求分类、轮播图、推荐商品和分类商品。商品列表页根据分类 id 请求商品列表，并支持价格筛选、销量排序、价格排序和分页。商品详情页根据商品 id 请求商品详情，用户可以选择数量并加入购物车。

## 11. 购物车模块怎么讲

购物车模块要求用户登录。用户在商品详情页点击加入购物车时，前端请求 `/cart/` 接口，把商品 id 和购买数量提交给后端。

后端购物车模型 `Cart` 关联用户和商品，并保存商品数量。后端在新增购物车时会判断当前用户是否已经添加过同一个商品，如果已经存在，就累加数量；如果不存在，就新增购物车记录。

购物车列表接口只返回当前登录用户自己的购物车数据，避免用户之间数据混乱。前端购物车页可以修改商品数量，修改后重新刷新购物车，并更新 Vuex 中的购物车总数和总价。

## 12. 订单模块怎么讲

订单模块是项目最核心的后端业务。

用户在结算页点击提交订单后，前端请求 `/order/` 接口，提交联系人、手机号、地址、支付方式和备注。后端会从当前用户的购物车中读取商品，创建订单主表 `Order`，再为每个购物车商品创建订单明细 `OrderGoods`。

后端创建订单时使用事务处理，并用 `select_for_update()` 给商品加锁，防止并发下库存出错。它会校验库存，扣减库存，增加销量，计算订单总数量和总金额，保存订单，然后清空购物车。

这部分可以重点讲：订单生成不是简单插入一条订单，而是包含库存校验、事务控制、订单主表、订单明细、价格快照和清空购物车。

## 13. 用户模块怎么讲

用户模型 `MyUser` 继承 Django 的 `AbstractUser`，在默认用户字段基础上扩展了真实姓名、手机号、性别、生日、头像、会员等级和状态等字段。

注册时，后端会校验用户名是否重复、手机号是否重复、手机号格式是否合法，并使用 `set_password()` 加密保存密码。

登录时，项目使用自定义认证后端，支持用户名或手机号登录。登录成功后返回 JWT token，前端后续请求通过 token 识别当前用户。

## 14. 地址模块怎么讲

地址模块用于管理用户收货地址。地址模型 `Address` 保存省份、城市、区域、详细地址、联系人、手机号、是否默认地址和所属用户。

新增地址时，前端不需要传用户 id，后端通过当前 JWT 登录用户自动绑定。地址列表接口只返回当前用户自己的地址。结算页会读取地址列表，并默认使用第一条地址作为收货信息。

## 15. 后端为什么用 Serializer

Serializer 的作用是把 Django 模型对象转换成前端可以使用的 JSON 数据，同时也可以校验前端提交的数据。

比如商品序列化器负责返回商品信息，用户注册序列化器负责校验用户名、密码和手机号，购物车序列化器负责处理同一个商品重复加入购物车时的数量累加。

## 16. 后端为什么用 ViewSet 和 Router

ViewSet 可以把增删改查逻辑集中写在一个类里，Router 可以自动生成常见 REST API 路由。比如 `CartViewset` 可以同时处理购物车列表、新增购物车、修改购物车和删除购物车。

这样比手写很多单独 URL 和视图函数更简洁，也更符合 RESTful 接口设计。

## 19. 可以主动说出的项目亮点

第一个亮点是项目采用前后端分离架构，前端页面和后端接口职责清晰。

第二个亮点是使用 JWT 实现登录认证，前端通过 Axios 拦截器统一携带 token，避免每个请求重复写认证逻辑。

第三个亮点是使用 Vuex 管理用户和购物车状态，让头部、购物车页和结算页能够共享购物车数量和总价。

第四个亮点是后端订单生成使用事务和数据库锁，能保证订单、库存和购物车处理的一致性。

第五个亮点是后端使用 DRF 的 ViewSet、Serializer、Router，提高了接口开发效率。

## 20. 可以主动说出的项目不足

第一，前端有一些备份文件和重复接口封装，比如 `checkout copy.vue`、`store copy.js`，项目结构可以进一步清理。

第二，前端部分路径写法不够规范，比如用户资料接口写成 `/users/1/`，虽然后端实际返回当前用户，但语义上容易误解。

第三，购物车和订单中都存在库存扣减逻辑，真实项目中更建议只在提交订单时统一扣减库存，避免重复扣库存。

第四，地址默认值更新时没有按当前用户过滤，真实项目中应该只修改当前用户自己的地址。

第五，配置文件里包含本地数据库账号等信息，正式项目应该使用环境变量管理敏感配置。

答辩时主动说不足反而是加分项，因为说明你不是只会跑项目，而是真的读懂了代码。

## 22. 如果老师问“登录怎么实现”

可以回答：

登录使用 JWT。用户在前端输入用户名和密码后，请求后端 `/login/` 接口。后端通过自定义认证后端支持用户名或手机号登录。登录成功后返回 token，前端把 token 保存到本地缓存，并把用户信息保存到 Vuex。之后访问购物车、订单、地址等接口时，Axios 请求拦截器会自动把 token 放到请求头中，后端根据 token 判断当前用户身份。

## 23. 如果老师问“购物车怎么实现”

可以回答：

购物车表关联用户和商品，并保存购买数量。用户加入购物车时，后端先判断当前用户是否已经添加过这个商品，如果已经有记录，就在原来的数量上累加；如果没有，就新增一条购物车记录。购物车列表接口只返回当前登录用户自己的购物车数据。前端通过 Vuex 保存购物车列表，并计算总数量和总价格。

## 24. 如果老师问“订单怎么实现”

可以回答：

订单提交时，前端提交收货人、手机号、地址、支付方式和备注。后端从当前登录用户的购物车中读取商品，创建订单主表，然后为每个购物车商品创建订单明细。创建过程中使用事务保证一致性，并通过数据库锁校验和扣减库存，同时增加商品销量。订单创建完成后，后端会清空当前用户购物车，并返回订单生成成功。

## 25. 如果老师问“商品列表怎么实现筛选和分页”

可以回答：

商品列表接口使用 Django REST Framework 和 django-filter。后端定义了商品过滤器，可以根据分类、最低价格、最高价格和名称筛选商品；同时使用分页类控制每页显示数量。前端列表页会把分类 id、价格范围、排序字段和页码作为参数传给 `/goods/` 接口，后端返回分页后的商品数据。
