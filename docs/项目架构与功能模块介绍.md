# 项目架构与功能模块介绍

## 1. 项目定位

本项目是一个基于前后端分离架构实现的综合实训商城系统，面向电商业务场景，覆盖用户从浏览商品、查看详情、加入购物车、提交订单到管理个人信息和收货地址的完整购物流程。

项目既可以作为课程实训、毕业设计或答辩展示项目，也可以作为学习 Vue 单页应用、Django REST Framework 接口开发、JWT 登录认证、购物车状态管理和订单事务处理的综合案例。

项目整体目标是通过一个相对完整的商城业务闭环，展示以下能力：

- 前端页面展示与交互能力
- 前后端分离开发能力
- REST API 设计与实现能力
- 用户登录认证与权限控制能力
- 商品、购物车、订单等典型电商业务建模能力
- 数据库事务、库存校验、订单明细生成等后端业务处理能力

## 2. 技术架构概览

项目由两个主要部分组成：

```text
myshop-vue/     Vue 2 前端项目
myshop-api2/    Django REST API 后端项目
```

前端负责页面展示、路由跳转、用户交互、状态管理和接口调用；后端负责业务数据建模、接口提供、登录认证、数据校验和订单处理。

### 2.1 前端技术栈

前端位于 `myshop-vue`，主要技术包括：

- Vue 2：构建单页应用页面
- Vue Router：管理首页、商品列表、详情页、购物车、用户中心等页面路由
- Vuex：保存登录用户信息、购物车列表、购物车数量和总价
- Axios：封装 HTTP 请求，与后端 REST API 通信
- Webpack：前端开发服务和生产构建
- vue-awesome-swiper：实现首页轮播等展示效果

前端核心入口：

```text
myshop-vue/src/main.js              Vue 应用入口
myshop-vue/src/App.vue              根组件
myshop-vue/src/router/index.js      前端路由配置
myshop-vue/src/vuex/store.js        全局状态管理
myshop-vue/src/api/                 业务接口封装
myshop-vue/src/utils/request.js     Axios 请求实例
myshop-vue/src/views/               页面组件
```

### 2.2 后端技术栈

后端位于 `myshop-api2`，主要技术包括：

- Django 3.1.5：后端 Web 框架
- Django REST Framework：构建 REST API
- djangorestframework-jwt：实现 JWT 登录认证
- django-filter：实现商品筛选、过滤等能力
- django-cors-headers：处理前后端跨域请求
- django-ckeditor：支持商品详情富文本内容
- SQLite / PostgreSQL：数据存储，项目支持通过 `DATABASE_URL` 配置数据库

后端核心入口：

```text
myshop-api2/manage.py               Django 管理入口
myshop-api2/myshop/settings.py      项目配置
myshop-api2/myshop/urls.py          后端总路由
myshop-api2/apps/goods/             商品模块
myshop-api2/apps/order/             购物车与订单模块
myshop-api2/apps/users/             用户模块
myshop-api2/apps/basic/             基础资料模块
myshop-api2/common/                 通用响应、权限、视图基类等
```

## 3. 整体架构说明

本项目采用典型的 B/S 架构和前后端分离模式。

```text
用户浏览器
   |
   | 访问页面、点击操作
   v
Vue 前端应用
   |
   | Axios 请求 REST API
   v
Django REST Framework 后端
   |
   | ORM 读写数据
   v
数据库
```

整体调用流程如下：

1. 用户在浏览器中访问前端页面。
2. Vue Router 根据路径展示对应页面，例如首页、商品列表、商品详情、购物车或个人中心。
3. 页面组件调用 `src/api` 中封装好的接口方法。
4. 接口方法通过 Axios 向 Django 后端发送请求。
5. 后端视图接收请求，调用模型、序列化器、权限和业务逻辑。
6. 后端返回统一格式的数据给前端。
7. 前端根据返回数据更新页面或 Vuex 全局状态。

## 4. 后端应用模块划分

后端业务主要划分为四个应用：`goods`、`order`、`users`、`basic`。

### 4.1 商品模块 goods

商品模块位于：

```text
myshop-api2/apps/goods/
```

主要负责商品分类、商品信息、首页轮播和首页商品展示。

核心模型包括：

- `GoodsCategory`：商品分类，支持父子分类结构，可以用于构建导航分类和商品列表分类筛选。
- `Goods`：商品主体信息，包括商品名称、分类、市场价、实际价格、库存、销量、点击数、收藏数、商品详情、主图、是否推荐等字段。
- `Slide`：首页轮播图，关联商品和轮播图片。

主要功能包括：

- 商品分类列表展示
- 商品分类详情查询
- 商品列表查询
- 商品详情查询
- 首页轮播图展示
- 首页分类商品展示
- 商品搜索、筛选、排序和分页
- 商品详情访问量统计

对应前端页面主要包括：

```text
myshop-vue/src/views/index/      首页相关组件
myshop-vue/src/views/list/       商品列表页
myshop-vue/src/views/detail/     商品详情页
myshop-vue/src/views/app/head.vue 分类导航和头部组件
```

### 4.2 用户模块 users

用户模块位于：

```text
myshop-api2/apps/users/
```

主要负责用户注册、登录认证、用户资料管理和会员信息扩展。

核心模型为：

- `MyUser`：自定义用户模型，继承 Django 的 `AbstractUser`，扩展了真实姓名、手机号、性别、生日、头像、会员等级、账号状态、创建时间和更新时间等字段。

主要功能包括：

- 用户注册
- 用户登录
- 用户名或手机号登录
- JWT Token 认证
- 当前登录用户资料查询
- 当前登录用户资料修改
- 用户状态和会员等级管理

对应前端页面主要包括：

```text
myshop-vue/src/views/user/login.vue     用户登录
myshop-vue/src/views/user/reg.vue       用户注册
myshop-vue/src/views/user/profile.vue   个人资料
```

登录成功后，前端会将 token 保存到本地，并将用户信息保存到 Vuex 中。后续访问购物车、订单、地址、用户资料等接口时，请求会携带：

```text
Authorization: JWT <token>
```

### 4.3 购物车与订单模块 order

购物车与订单模块位于：

```text
myshop-api2/apps/order/
```

该模块是商城项目中最能体现业务流程的部分，负责购物车管理、订单生成、订单商品明细和库存处理。

核心模型包括：

- `Cart`：购物车模型，记录用户、商品和购买数量。
- `Order`：订单主表，记录订单号、商品总件数、订单总金额、收货地址、联系人、联系电话、支付方式、订单备注、订单状态和下单用户。
- `OrderGoods`：订单商品明细表，记录订单与商品的关系、购买数量和下单时的商品价格快照。

主要功能包括：

- 加入购物车
- 查询购物车
- 修改购物车商品数量
- 删除购物车商品
- 计算购物车总数量和总价
- 提交订单
- 订单库存校验
- 订单商品明细生成
- 下单后清空购物车
- 查询我的订单

对应前端页面主要包括：

```text
myshop-vue/src/views/cart/cart.vue       购物车页面
myshop-vue/src/views/cart/checkout.vue   结算页面
myshop-vue/src/views/user/myorder.vue    我的订单页面
```

订单创建流程大致如下：

```text
用户提交订单
  -> 后端读取当前用户购物车
  -> 创建订单主表
  -> 校验商品库存
  -> 扣减库存并增加销量
  -> 生成订单商品明细
  -> 统计订单总数量和总金额
  -> 清空购物车
  -> 返回订单创建结果
```

该流程体现了电商系统中比较典型的主从表写入、库存校验、价格快照和事务处理思想。

### 4.4 基础资料模块 basic

基础资料模块位于：

```text
myshop-api2/apps/basic/
```

当前主要负责用户收货地址管理。

核心模型为：

- `Address`：用户配送地址模型，包含省份、城市、区域、详细地址、联系人、联系电话、所属用户、是否默认地址和创建时间。

主要功能包括：

- 查询当前用户收货地址
- 新增收货地址
- 修改收货地址
- 删除收货地址
- 设置默认收货地址

对应前端页面主要包括：

```text
myshop-vue/src/views/user/address.vue
```

## 6. 核心业务流程

### 6.1 用户购物流程

```text
进入首页
  -> 浏览分类和推荐商品
  -> 进入商品列表
  -> 查看商品详情
  -> 加入购物车
  -> 登录或注册
  -> 进入购物车
  -> 结算并选择地址
  -> 提交订单
  -> 查看我的订单
```

该流程覆盖了一个商城项目最基本的用户购买闭环。

### 6.2 登录认证流程

```text
用户输入用户名或手机号和密码
  -> 前端请求 /login/
  -> 后端进行用户认证
  -> 登录成功后返回 JWT token
  -> 前端保存 token 和用户信息
  -> 后续请求在请求头中携带 JWT token
  -> 后端识别当前登录用户
```

该流程实现了前后端分离项目中常见的无状态登录认证方式。

### 6.3 订单生成流程

```text
前端提交订单信息
  -> 后端获取当前用户购物车
  -> 创建订单主表
  -> 遍历购物车商品
  -> 校验库存
  -> 生成订单商品明细
  -> 更新商品库存和销量
  -> 更新订单总金额和总数量
  -> 删除购物车记录
  -> 返回订单生成成功
```

该流程体现了电商后端中比较关键的事务处理和数据一致性设计。

## 7. 数据模型关系概览

项目核心数据关系可以概括为：

```text
用户 MyUser
  ├── 收货地址 Address
  ├── 购物车 Cart
  └── 订单 Order

商品分类 GoodsCategory
  └── 商品 Goods
        ├── 首页轮播 Slide
        ├── 购物车 Cart
        └── 订单明细 OrderGoods

订单 Order
  └── 订单明细 OrderGoods
```

主要关系说明：

- 一个用户可以拥有多个收货地址。
- 一个用户可以拥有多条购物车记录。
- 一个用户可以拥有多个订单。
- 一个商品分类可以包含多个商品。
- 一个商品可以出现在购物车和订单明细中。
- 一个订单可以包含多条订单商品明细。

## 8. 项目亮点

### 8.1 前后端分离清晰

前端负责页面与交互，后端负责数据与业务逻辑，两者通过 REST API 通信，结构清晰，适合作为前后端分离项目展示。

### 8.2 业务闭环完整

项目覆盖首页、分类、商品详情、购物车、结算、订单、登录注册、地址管理和个人中心，具备完整商城系统的基本业务闭环。

### 8.3 后端模块划分明确

后端按照业务领域拆分为商品、订单、用户、基础资料等应用，每个应用内部包含模型、序列化器、视图和路由，符合 Django 项目的常见组织方式。

### 8.4 使用 JWT 完成登录认证

项目采用 JWT 作为前后端分离认证方案，登录成功后由前端保存 token，后续请求携带 token 访问需要登录的接口。

### 8.5 订单流程具备真实业务特征

订单模块包含订单主表、订单明细、购物车转订单、库存校验、销量更新和购物车清空等逻辑，能够体现真实电商业务中的关键处理流程。

### 8.6 前端状态管理体现全局数据共享

项目通过 Vuex 管理用户信息和购物车数据，使头部购物车数量、购物车页面和结算页面能够共享同一份状态。

## 9. 展示时可以重点说明的内容

如果用于答辩、汇报或项目展示，可以重点介绍以下内容：

1. 项目是一个前后端分离商城系统，前端使用 Vue 2，后端使用 Django REST Framework。
2. 项目实现了用户从浏览商品到下单的完整业务闭环。
3. 后端按照商品、用户、购物车订单、地址等业务模块拆分，结构清晰。
4. 登录认证使用 JWT，适合前后端分离场景。
5. 购物车和订单模块体现了电商系统中典型的数据流转过程。
6. 订单生成中涉及库存校验、订单主从表写入、价格快照和购物车清空。
7. 前端通过 Vue Router 管理页面，通过 Vuex 管理登录用户和购物车状态，通过 Axios 调用后端接口。
8. 项目支持本地开发和生产部署，可以通过 Conda 管理 Python 环境，并通过 npm 构建前端资源。
