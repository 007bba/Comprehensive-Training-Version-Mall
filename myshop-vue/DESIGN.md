# DESIGN.md — 特产电商商城 · 首页设计规范

> 品牌气质：温暖、亲切、有品质感、不廉价、东方美学
> 参考调性：「一条」「物道」——干净、留白、有温度、内容感强
> 技术栈：Vue 2 + scoped CSS

本文件只定义**设计规则**，不含代码。所有组件实现必须以此为唯一依据。

---

## 0. 设计基调（一句话原则）

> **「米纸为底，朱墨点睛」**——大面积温润的米白留白，文字如墨，唯有价格与行动点用一抹克制的朱砂红。
> 宁可少，不可廉价。每多一个颜色、一道阴影、一个圆角，都要问：它让页面更贵了，还是更吵了？

---

## 1. 色彩方案

强调**低饱和、暖调、东方矿物色**。不用纯黑、不用纯白、不用蓝紫科技感。

### 主色（Primary · 点睛用，占比 < 5%）
| 名称 | HEX | 用途 |
|------|------|------|
| 朱砂红 `--brand` | `#B23A2E` | 价格、主 CTA、关键强调。源自传统朱砂/印泥，暖而沉，不刺眼 |
| 朱砂深 `--brand-dark` | `#993023` | 按钮 hover / 按下态 |
| 朱砂浅 `--brand-soft` | `#F4E3DE` | 标签底、选中态浅背景 |

### 辅色（Secondary · 区分场景，少量使用）
| 名称 | HEX | 用途 |
|------|------|------|
| 黛茶绿 `--accent` | `#5C6B5A` | 「产地 / 自然 / 新鲜」类标签、次级链接点缀 |
| 陈茶褐 `--accent-warm` | `#9C7A53` | 「手作 / 传统 / 时令」类点缀、分隔装饰 |

### 中性色（Neutral · 页面 90% 由它构成）
| 名称 | HEX | 用途 |
|------|------|------|
| 米纸白 `--bg` | `#FAF7F1` | 页面背景（**不是纯白**，带暖意） |
| 卡片白 `--surface` | `#FFFFFF` | 卡片、浮层底 |
| 浅米线 `--line` | `#ECE6DC` | 分隔线、卡片描边（替代灰线，更柔） |
| 墨 `--ink` | `#2B2622` | 主标题、正文（**暖黑，非 #000**） |
| 次墨 `--ink-2` | `#6E665E` | 正文辅助、副标题 |
| 远墨 `--ink-3` | `#A89F94` | 占位、原价删除线、弱提示 |

### 功能色（Functional · 仅状态用）
| 名称 | HEX | 用途 |
|------|------|------|
| 成功 `--success` | `#5C6B5A` | 复用黛茶绿，保持克制（加入成功提示） |
| 警示 `--warn` | `#C08A2D` | 库存紧张、限时 |
| 错误 `--error` | `#B23A2E` | 复用朱砂，不另造红 |

> **配色比例参考**：中性≈90% · 辅色≈5% · 主色≈5%。

## 2. 字体系统

中文优先，衬线用于「内容/品质感」标题，无衬线用于功能性文字，形成东方杂志式的对比。

### 字体族
```
标题/品牌（衬线，有温度）：
  "Noto Serif SC", "Songti SC", "STSong", serif
正文/功能（无衬线，清晰）：
  -apple-system, "PingFang SC", "Hiragino Sans GB",
  "Microsoft YaHei", sans-serif
```
> 仅楼层大标题、商品名可用衬线；价格、按钮、标签等功能文字用无衬线。

### 字号 / 字重 / 行高
| 层级 | 字号 | 字重 | 行高 | 字距 | 场景 |
|------|------|------|------|------|------|
| 楼层标题 H2 | 24px | 600 | 1.4 | 1px | 「时令鲜货」等 section 标题（衬线） |
| 楼层副标题 | 13px | 400 | 1.5 | 2px | 标题下一行英文/拼音点缀，用 `--ink-3` |
| 卡片商品名 | 15px | 500 | 1.6 | 0 | 最多两行（无衬线） |
| 价格 | 19px | 700 | 1.2 | 0 | 朱砂红，¥ 符号缩小到 13px |
| 原价（删除） | 12px | 400 | 1.2 | 0 | `--ink-3`，删除线 |
| 标签 / 辅助 | 12px | 400 | 1.4 | 0.5px | 产地、规格等 |
| 按钮文字 | 13px | 500 | 1 | 1px | — |

> 价格的 `¥` 用 `<small>` 缩小并下沉，营造杂志价签质感，不要和数字同号。

## 3. 间距规则

基准单位 **4px**，常用刻度：`4 / 8 / 12 / 16 / 24 / 32 / 48 / 64`。

| 场景 | 值 |
|------|------|
| 卡片内边距 | 16px |
| 卡片内元素纵向间隔（图→名→价） | 12px |
| 网格列间距（gutter） | 24px |
| 网格行间距 | 32px |
| 楼层标题与内容 | 24px |
| 楼层之间 | 48px |
| 内容区左右安全边距 | 16px（容器 max-width 1200px，居中） |

> 留白原则：**宁多勿挤**。商品之间的呼吸感比「一屏塞更多货」更重要——这是品质感的核心来源。

## 4. 卡片设计规范

| 属性 | 规则 |
|------|------|
| 背景 | `--surface`（#FFF） |
| 圆角 | 外卡片 14px，图片容器 10px |
| 描边 | 1px `--line`（**用柔描边代替重阴影**，更克制、更东方） |
| 阴影（静止） | `0 2px 8px rgba(43, 38, 34, 0.04)`——极轻，几乎只是「离开纸面一点点」 |
| 内边距 | 16px |
| 图片比例 | 1:1（`object-fit: cover`），底色 `--bg` 防止空白闪烁 |
| 商品名 | 两行截断，固定最小高度避免错位 |

> 禁止重投影、彩色阴影、内发光。阴影只用暖黑低透明度。

## 5. 按钮规范

### 主按钮（加入购物车 / 去结算）
| 属性 | 规则 |
|------|------|
| 背景 | `--brand` 朱砂红 |
| 文字 | #FFF，13px / 500 |
| 圆角 | 22px（胶囊）或图标按钮取 50% 圆形 |
| 高度 | 标准 40px；卡片内图标按钮 36×36px |
| 内边距 | 文字按钮左右 20px |
| hover | 背景 → `--brand-dark`，**不放大、不发光**，仅颜色加深 + 轻微上浮可选 |
| 按下 | 颜色 `--brand-dark`，scale 不超过 0.98 |

### 次按钮 / 文字链接
| 属性 | 规则 |
|------|------|
| 背景 | 透明或 `--brand-soft` |
| 文字 | `--brand` 或 `--ink-2` |
| 描边 | 1px `--line` 或 `--brand`（描边款） |
| hover | 文字/描边转 `--brand`，背景转 `--brand-soft` |

> 一个卡片内**只允许一个主按钮**。CTA 不抢标题、不抢价格的视线，三者形成「看 → 心动 → 行动」的顺序。

## 6. 动效原则

克制、自然、像翻纸而非弹窗。

| 项目 | 规则 |
|------|------|
| 默认过渡时长 | 200ms |
| 大元素 / 浮层 | 280–320ms |
| 缓动函数 | `cubic-bezier(0.22, 0.61, 0.36, 1)`（ease-out 风，进场利落、收尾柔） |
| 卡片 hover | 图片轻微放大（scale 1.03，**不超过 1.05**）+ 描边转暖；上浮 ≤ 2px |
| 按钮 hover | 仅颜色过渡，禁止 bounce / 阴影膨胀 |
| 禁止 | 弹跳、旋转、闪烁、视差滥用、入场时全员飞入 |

> 动效的目的是「确认交互」，不是「表演」。用户察觉不到动效存在，但缺了会觉得生硬——这就对了。

---

## 7. 禁止出现的 AI 模板套路（硬性红线）

- ❌ 渐变紫 / 蓝紫科技渐变 / 霓虹色
- ❌ 毛玻璃（backdrop-filter blur）泛滥
- ❌ 纯黑 #000 文字、纯白 #FFF 背景大面积直铺
- ❌ 高饱和原色（纯红 #FF0000、亮绿、电光蓝）
- ❌ 重投影 / 彩色发光阴影 / 多层 box-shadow 堆叠
- ❌ 大圆角泡泡风（圆角 > 24px 的卡片）
- ❌ emoji 当图标、彩色 emoji 装饰
- ❌ 渐变文字、描边文字、文字阴影
- ❌ 居中对齐一切（标题、正文应有左对齐的版面骨架）
- ❌ 每个元素都加动画的「全员入场秀」
- ❌ 千篇一律的「卡片 hover 整体放大 + 紫色光晕」

> 检验标准：把这个页面截图，**不能让人一眼看出是 AI 默认生成的**。它应该像一本讲究的中文生活杂志的电商版。

---

## 8. 复用约定

- 所有颜色、圆角、间距应以 CSS 变量在组件根定义（`--brand` 等），便于全站统一。
- 每个组件 scoped，但变量命名跨组件保持一致。
- 新组件实现前先回查本文件，不得自创色值与圆角。
